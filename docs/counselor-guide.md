## 系统概述

### 星职助手的三种角色

| 角色 | 权限说明 |
|------|---------|
| **user**（普通用户） | 心智障碍人士，使用任务引导、情绪记录、AI 陪伴等功能 |
| **counselor**（辅导员） | 查看和管理用户、创建任务模板、查看统计数据 |
| **admin**（管理员） | 拥有所有权限，包括修改用户角色、删除模板等 |

### 辅导员可以做什么

- 查看所有用户的列表和详情
- 创建和管理任务模板
- 查看全局统计数据（用户数、任务完成数、情绪分布等）
- 查看单个用户的详细数据（任务完成情况、情绪趋势、成就进度）
- 查看情绪统计报表，辅助辅导决策

## 账号注册与登录

### 注册辅导员账号

辅导员账号需要由 **管理员** 创建并分配角色。流程如下：

1. 先使用普通用户注册流程注册账号（手机号 + 姓名 + 密码）
2. 联系管理员，提供注册时使用的手机号
3. 管理员在管理后台将该账号的角色修改为 `counselor`

> **注意**：出于安全考虑，系统不支持用户自行注册时选择辅导员角色。

### 登录管理后台

1. 打开浏览器，访问管理后台地址（默认为 `frontend/admin.html`）
2. 输入管理员分配的手机号和密码
3. 点击「登录」按钮
4. 登录成功后进入管理后台首页

### 管理后台入口

| 入口 | 地址 | 说明 |
|------|------|------|
| 管理后台 | `frontend/admin.html` | 辅导员/管理员专用后台 |
| 用户端 | `frontend/index.html` | 普通用户使用的应用 |
| API 文档 | `http://localhost:8000/docs` | Swagger 接口文档 |

## 用户管理

### 查看用户列表

1. 点击左侧菜单 **「用户管理」**
2. 页面显示所有注册用户的列表
3. 每个用户显示以下信息：
   - 姓名
   - 手机号
   - 角色
   - 注册时间
   - 已完成任务数
   - 情绪记录数

### 搜索和筛选

| 操作 | 说明 |
|------|------|
| 关键词搜索 | 输入姓名或手机号进行搜索 |
| 角色筛选 | 按角色（user/counselor/admin）筛选 |
| 分页浏览 | 每页显示 20 条，可翻页查看 |

### 查看用户详情

点击某个用户，进入该用户的详情页面，包含：

#### 基本信息
- 姓名、手机号、角色、注册时间

#### 统计数据
- 总任务数 / 已完成任务数
- 情绪记录总数
- 成就解锁数

#### 近 7 天趋势
- 每日完成任务数
- 每日获得星星数
- 每日情绪记录数

#### 最近情绪记录
- 显示该用户最近 10 条情绪记录
- 包含情绪类型、时间、AI 建议小贴士

### 修改用户角色（仅管理员）

> **注意**：此功能仅管理员可用。

1. 在用户详情页，找到 **「修改角色」** 按钮
2. 选择新的角色：
   - `user` - 普通用户
   - `counselor` - 辅导员
   - `admin` - 管理员
3. 确认修改

## 任务模板管理

任务模板是系统的核心功能。辅导员可以为用户创建适合的任务模板，将复杂工作拆解为简单步骤。

### 查看任务模板列表

1. 点击左侧菜单 **「任务模板」**
2. 页面显示所有任务模板
3. 每个模板显示：
   - 模板名称
   - 图标
   - 步骤数
   - 是否公开
   - 使用次数
   - 创建时间

### 创建新任务模板

1. 点击 **「创建模板」** 按钮
2. 填写模板信息：

   | 字段 | 说明 | 示例 |
   |------|------|------|
   | 名称 | 任务模板的名称 | "整理文件" |
   | 图标 | 任务的图标标识 | "📁" |
   | 步骤 | 任务拆解的步骤列表 | 见下方示例 |
   | 是否公开 | 是否对所有用户可见 | 是 |

3. **步骤编写示例**：

   ```json
   [
     "找到需要整理的文件",
     "把文件按类别分组",
     "把每一类文件放进对应的文件夹",
     "在文件夹上贴好标签",
     "把文件夹放回文件柜"
   ]
   ```

4. 点击 **「保存」** 完成创建

### 步骤编写最佳实践

编写任务步骤时，请遵循以下原则：

1. **简单明确**：每个步骤只做一件事
   - 好："把文件放进文件夹"
   - 不好："整理好所有文件并分类"

2. **使用动词开头**：让用户知道要做什么
   - 好："打开电脑"
   - 不好："电脑"

3. **步骤数量适中**：建议 3-8 个步骤
   - 太少（1-2 步）：可能拆解不够细
   - 太多（10+ 步）：可能让用户感到压力

4. **按顺序排列**：步骤应该按实际操作顺序排列

5. **考虑用户能力**：根据用户的能力水平调整步骤的复杂度

### 编辑和删除模板

| 操作 | 权限 | 说明 |
|------|------|------|
| 编辑模板 | 管理员 | 修改模板名称、图标、步骤等 |
| 删除模板 | 管理员 | 删除模板（有进行中任务的模板无法删除） |
| 设置公开/私有 | 辅导员+ | 控制模板是否对所有用户可见 |

## 数据解读与分析

### 全局统计

在仪表盘或统计页面，可以查看以下全局数据：

#### 核心指标

| 指标 | 意义 | 关注点 |
|------|------|--------|
| 总用户数 | 平台使用规模 | 持续增长说明推广有效 |
| 今日活跃用户 | 用户参与度 | 活跃率 = 活跃用户 / 总用户 |
| 今日完成任务数 | 用户产出 | 反映整体任务完成情况 |
| 今日情绪记录数 | 情绪关注程度 | 记录越多，情绪管理越好 |

#### 趋势分析

- **近 7 天趋势图**：展示每日任务完成数、星星数、情绪记录数的变化
- **用户注册趋势**：展示近 30 天每日新增用户数

### 用户个人数据分析

在用户详情页，重点关注以下数据：

#### 任务完成情况

| 数据 | 解读 |
|------|------|
| 完成率高（>80%） | 用户能力匹配良好，可考虑提升难度 |
| 完成率中等（50-80%） | 基本适应，保持当前难度 |
| 完成率低（<50%） | 可能需要降低难度或增加辅导 |

#### 每日趋势

观察用户近 7 天的数据变化：

- **任务完成数上升**：用户在进步，给予鼓励
- **任务完成数下降**：可能遇到困难，需要关注
- **情绪记录减少**：可能使用意愿降低，主动联系
- **星星数增加**：正向反馈，继续保持

## 情绪趋势分析

情绪数据是辅导决策的重要参考。

### 查看情绪统计

1. 点击左侧菜单 **「情绪统计」**
2. 选择统计时间范围（默认近 30 天）
3. 查看以下数据：

### 情绪分布

系统会统计各情绪类型的占比：

| 情绪类型 | 正常范围 | 需要关注 |
|---------|---------|---------|
| 😄 开心 | > 30% | - |
| 😌 平静 | > 25% | - |
| 😰 焦虑 | < 20% | 持续 > 30% 需关注 |
| 😢 难过 | < 15% | 持续 > 25% 需关注 |
| 😤 生气 | < 10% | 持续 > 20% 需关注 |

### 情绪趋势图

趋势图展示每天各情绪类型的数量变化，重点关注：

1. **负面情绪持续上升**：连续 3 天以上焦虑/难过/生气增加
   - 建议：主动与用户沟通，了解原因
   - 考虑调整任务难度

2. **正面情绪稳定**：开心/平静占主导
   - 说明：用户状态良好
   - 可以适当增加挑战

3. **情绪波动大**：一天内多次情绪变化
   - 建议：关注工作环境是否有刺激因素
   - 考虑增加休息频率

### 辅导建议

根据情绪数据，可以采取以下措施：

| 情况 | 建议措施 |
|------|---------|
| 用户长期焦虑 | 降低任务难度，增加鼓励频率，安排更多休息 |
| 用户突然情绪低落 | 主动沟通，了解原因，必要时调整工作内容 |
| 用户情绪稳定向好 | 给予正向反馈，适当提升任务难度 |
| 用户很少记录情绪 | 提醒记录的重要性，但不强制 |

## API 接口说明

以下接口供技术对接使用。所有接口均需 JWT Token 认证。

### 认证方式

在请求头中携带 Token：

```
Authorization: Bearer <your-jwt-token>
```

### 用户管理接口

#### 获取用户列表

```
GET /api/admin/users?page=1&page_size=20&role=user&keyword=张三
```

**权限**：counselor / admin

**参数**：

| 参数 | 类型 | 说明 |
|------|------|------|
| page | int | 页码，默认 1 |
| page_size | int | 每页数量，默认 20，最大 100 |
| role | string | 角色筛选：user / counselor / admin |
| keyword | string | 搜索关键词（姓名/手机号） |

**响应示例**：

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "total": 50,
    "page": 1,
    "page_size": 20,
    "items": [
      {
        "id": 1,
        "phone": "13800138001",
        "name": "张三",
        "role": "user",
        "avatar": null,
        "created_at": "2025-01-15T10:30:00",
        "tasks_completed": 12,
        "mood_count": 25
      }
    ]
  }
}
```

#### 获取用户详情

```
GET /api/admin/users/{user_id}
```

**权限**：counselor / admin

**响应包含**：用户基本信息、统计数据、近 7 天趋势、最近情绪记录。

### 任务模板接口

#### 创建任务模板

```
POST /api/admin/task-templates
```

**权限**：counselor / admin

**请求体**：

```json
{
  "name": "整理文件",
  "icon": "📁",
  "steps": [
    "找到需要整理的文件",
    "把文件按类别分组",
    "把文件放进对应的文件夹",
    "在文件夹上贴好标签",
    "把文件夹放回文件柜"
  ],
  "is_public": true
}
```

#### 获取任务模板列表

```
GET /api/admin/task-templates?page=1&page_size=20&is_public=true
```

**权限**：counselor / admin

#### 更新任务模板

```
PUT /api/admin/task-templates/{template_id}
```

**权限**：admin

#### 删除任务模板

```
DELETE /api/admin/task-templates/{template_id}
```

**权限**：admin

> **注意**：有进行中任务的模板无法删除。

### 统计接口

#### 全局统计

```
GET /api/admin/stats
```

**权限**：counselor / admin

**响应包含**：总用户数、总完成任务数、总情绪记录数、总成就数、今日统计、近 7 天趋势、用户注册趋势。

#### 情绪统计

```
GET /api/admin/mood-stats?days=30
```

**权限**：counselor / admin

**参数**：

| 参数 | 类型 | 说明 |
|------|------|------|
| days | int | 统计天数，默认 30，最大 365 |

**响应包含**：总记录数、各情绪类型分布（含百分比）、每日情绪趋势。

### AI 接口

#### AI 聊天陪伴

```
POST /api/ai/chat
```

**请求体**：

```json

  "message": "我今天有点紧张",

```


#### 获取难度建议

```

```

## 最佳实践

### 任务模板设计建议

1. **从简单开始**：新用户先分配步骤少、操作简单的任务
2. **循序渐进**：根据用户进步情况逐步增加任务难度
3. **个性化定制**：根据每个用户的能力和岗位定制任务模板
4. **定期更新**：根据实际工作内容变化更新任务模板

### 用户辅导建议

1. **每日关注**：每天查看用户的数据变化，及时发现异常
2. **情绪优先**：情绪问题优先处理，情绪稳定才能有效工作
3. **正向激励**：多关注用户的进步，及时给予肯定和鼓励
4. **家校联动**：定期与用户家属沟通，形成支持网络

### 数据使用建议

1. **趋势比单点重要**：关注数据变化趋势，而非单日数据
2. **结合实际情况**：数据仅供参考，要结合实际观察和沟通
3. **保护隐私**：用户数据仅用于辅导目的，不得泄露
4. **定期总结**：每周/每月做一次数据总结，调整辅导策略
